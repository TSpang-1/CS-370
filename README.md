Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?

For this project, I was given the basic structure of the pirate treasure hunt game, which included the maze environment, supporting classes, and starter code for training the intelligent agent. My goal was completing the reinforcement learning portion of the project by implementing the deep Q-learning algorithm. I wrote the code that allowed the pirate agent to explore the maze, store experiences, train the neural network from those experiences, balance exploration and exploitation, and gradually improve its decisions over time. After training, the agent was able to consistently find the treasure by choosing the best path through the maze.

What do computer scientists do and why does it matter?

Computer scientists solve problems by designing software, algorithms, and systems that make everyday tasks easier and more efficient. They work on everything from mobile apps and websites to artificial intelligence and cybersecurity. Their work matters because technology is a part of pretty much every industry today, and well-designed software can help businesses to work more effectively while also making people's lives easier.

How do I approach a problem as a computer scientist?

Throughout this course, I learned that solving a problem starts with understanding exactly what needs to be accomplished before actually writing any code. I try to break a large problem down into smaller pieces, learn how each part works, and then build the solution one step at a time. During this project, I spent time understanding how reinforcement learning worked before making any changes to the code itself. Testing the agent, reviewing the results, and making adjustments helped me to improve the final solution.

What are my ethical responsibilities to the end user and the organization?

I believe developers have a responsibility to create software that is reliable, fair, and safe for the people who use it. AI systems should be tested carefully to lower any risk of errors and avoid unintended bias whenever possible. Developers should also respect user privacy, protect sensitive information, and be honest about what an AI system can and cannot do. Building software that users and organizations can trust is just as important as making it work correctly.
